librechat-agent

A small terminal coding agent for LLMs hosted behind LibreChat. A Python ReAct loop talks to LibreChat's webui REST API over your existing browser session — no provider API key — and can read, write, and edit files, run shell commands, and stream the model's answer and reasoning trace live into your terminal.

the idea

Most coding agents want a provider API key. But plenty of useful models live behind an org's LibreChat instance — reachable only through the same SSO-gated webui your browser already uses. librechat-agent skips the key entirely: it rides your existing LibreChat session and drives the model through the webui's own REST API, so any model the chat UI can reach becomes a tool-using terminal agent in your current directory.

It calls no LLM provider directly. Every model call goes agent → LibreChat → whatever endpoint LibreChat is configured to use. For local testing it can also talk straight to any OpenAI-compatible endpoint (e.g. Ollama), but LibreChat is the point.

how it works

  • ReAct loop — tool calling uses ReAct-style prompting with <tool_call> JSON blocks (Qwen2.5's native format), so it works even when the backend doesn't expose OpenAI tool definitions. Lenient fallback parsers catch models that drift into bare-JSON or TOOL_NAME: … styles.
  • Two-step protocol — each turn is a two-step exchange against LibreChat's internal API: POST /api/agents/chat/<endpoint> returns a streamId, then GET /api/agents/chat/stream/<streamId> delivers the response as server-sent events.
  • Session-cookie / JWT auth — no API key. Every request carries a browser-like User-Agent, an Authorization: Bearer <jwt> access token (~15 min), and the long-lived refreshToken cookie. On a 401 the agent auto-mints a fresh JWT via /api/auth/refresh.
  • Server-side state — LibreChat keeps the conversation itself; the agent tracks conversationId / parentMessageId so resume continues the same server-side conversation. Transient 429/5xx/timeout failures are retried with bounded backoff.
  • Three login modes — a real Chromium window for SSO/MFA, an email+password POST to /api/auth/login, or pasting the refreshToken cookie from your browser's devtools.

the terminal ui

  • Live streaming — responses render token-by-token; reasoning models get a separate dimmed ── thinking ── section before the ── answer ──.
  • Thinking recall — press Ctrl+T (or /thinking) to re-show the last captured reasoning trace, kept per chat.
  • Type while it works — input typed while the agent is busy is queued ([queued #1: …]) and auto-sent in order at the next turn boundary; manage it with /queue, /recall, or the Up arrow.
  • Background jobs — long commands run detached with log files; press Ctrl+B to background an already-running foreground command, or Ctrl+J for the interactive jobs menu.
  • Slash commands — /jobs, /kill, /thinking, /mode, /log, /name, /resume-friendly chat management, and more.

tools & safety

  • Real execution — run_bash runs actual shell commands (foreground or detached), read_file / list_dir / write_file / edit_file touch real files. Writes show a colored unified diff before asking permission.
  • Permission modes — safe (default) confirms anything destructive, auto-edit auto-allows file edits but asks before shell, yolo removes all guardrails — switchable mid-chat with /mode.
  • Bounded by design — foreground commands are killed after 60 s, tool output sent back to the model is capped (~2000 lines / 200 KB) with a pointer to the full log, and a single turn allows at most 20 chained tool calls.
  • TLS verification is disabled for LibreChat requests, because internal instances commonly use private CAs — so don't point it at hosts you don't trust.
  • Live credentials on disk — session.json holds cookies + JWT in ~/.config/cli-agent/; treat it like a password.

cli-agent — session
> create hello.py that prints "hi", then run it

── thinking ──
need a file then a run; write_file, then run_bash…

── answer ──
  [tool] write_file({'path': 'hello.py',
              'content': 'print("hi")'})
[ ok ] diff applied · hello.py

  [tool] run_bash({'command': 'python3 hello.py'})
Output: hi

Done — hello.py is written and prints "hi".

slash & keys

  • /jobs · /kill <id> — list / kill background jobs.
  • /thinking — last reasoning trace (also Ctrl+T).
  • /mode safe|auto-edit|yolo — switch permissions mid-chat.
  • /queue · /recall — manage queued input.
  • Ctrl+B — background the running command.
  • Ctrl+J — interactive jobs menu.
  • Ctrl+O — full log of the last command.
