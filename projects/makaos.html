makaos

A 64-bit operating system for x86-64, built from scratch: a two-stage NASM bootloader, a C kernel with preemptive scheduling and ring-3 userspace, a small libc, and an ext2 root filesystem — booted and tested in QEMU.

the idea

The fastest way to understand a machine is to write the software that owns it end to end. MakaOS starts at a 512-byte boot sector and climbs all the way up to ring-3 programs making syscalls — no firmware shortcuts, no borrowed kernel.

boot chain

  • Stage 1 — 512-byte MBR boot sector, loads stage 2 via INT 13h LBA reads.
  • Stage 2 — collects the E820 memory map and VBE video info, switches real → protected → long mode.
  • Kernel loader — freestanding C with its own AHCI/ATA driver, early PMM/VMM, higher-half handoff.
  • UEFI path — a clang-built BOOTX64.EFI on a GPT disk (FAT32 ESP + ext2), on the test branch.

kernel

  • Memory — buddy-allocator PMM from the E820 map, demand-paged VMM with per-process VMAs, higher-half mapping.
  • Scheduling — preemptive, PIT-driven, processes and kernel threads, context switch in assembly.
  • Userspace — ELF64 loader, syscall/sysret fast path via STAR/LSTAR MSRs, per-process kernel stacks.
  • 29 syscalls — fork/exec/waitpid, pipe, dup/dup2, signals, brk, full path & directory calls.
  • Storage & FS — PCI enumeration, IRQ-driven AHCI, ext2 read and write + FAT32 behind a VFS.
  • Shell — PS/2 keyboard and a VGA text-mode shell: ls, cat, run, edit, ps, …

where it's going

The test branch carries a ports tree — cross-compiled bash, foot, SDL3, libinput, libseat, and wlroots — working toward a Wayland desktop running on an OS that began life as 512 bytes of boot sector.

[ status ]

In active development. main boots reliably under QEMU and passes its POSIX suite, but this is a learning project: single-core, fixed disk layout, no networking yet, UEFI and desktop still stabilizing on side branches.
