makaos

A 64-bit operating system for x86-64, built from scratch: a UEFI bootloader, a 4-way SMP kernel with ~118 Linux-style syscalls, a from-scratch libc running bash, a TCP/IP stack over virtio-net, and a ported wlroots/Wayland desktop, all booted and tested in QEMU.

the idea

The fastest way to understand a machine is to write the software that owns it end to end. MakaOS climbs from the UEFI handoff all the way up to a multithreaded ring-3 userland and a Wayland compositor, no firmware shortcuts, no borrowed kernel.

boot

  • UEFI / OVMF, a clang-built COFF BOOTX64.EFI collects the E820 memory map and the GOP framebuffer, installs a higher-half mapping, and hands off to the kernel.
  • Legacy loader, a freestanding second-stage C loader with its own AHCI/ATA driver and early PMM/VMM is kept in-tree as the BIOS path.
  • Disk, a GPT image with a FAT32 ESP and an ext2 root, assembled by build.sh.

kernel

  • 4-way SMP, per-CPU runqueues, an MLFQ scheduler over a Chase-Lev work-stealing deque, RCU, and lock-minimized hot paths.
  • Memory, buddy-allocator PMM with per-CPU magazines, a demand-paging VMM with per-process VMAs, copy-on-write fork, and a slab + kernel-heap allocator.
  • Userspace, ELF64 loader, syscall/sysret fast path, per-process kernel stacks via the TSS, real TLS (%fs), futexes, and kernel-owned thread join, enough to run pthreads-heavy programs.
  • ~118 syscalls, Linux-style -errno returns: POSIX signals, pipe, dup/fcntl, epoll/eventfd/timerfd/signalfd, io_uring, and Unix-domain sockets with SCM_RIGHTS.
  • Drivers, PCI, IRQ-driven AHCI, NVMe, virtio-gpu (DRM/KMS), virtio-net, virtio-input (absolute tablet), Intel HD Audio, and PS/2 keyboard + mouse.
  • Networking & FS, an ARP / IPv4 / UDP / TCP stack over virtio-net, and ext2 read and write behind a small VFS with pollable /dev character devices.

userland

  • libc from scratch (System V AMD64 ABI) with TLS, pthreads, and a __thread errno; bash 5.2 runs as /bin/sh.
  • A POSIX conformance suite (test_posix1, 156 assertions) plus assorted apps, including a DOOM port.

desktop (the work in progress)

A wlroots-0.18 stack is cross-compiled and run on the kernel's virtio-gpu/DRM + libinput path: the sway compositor, the foot terminal, swaybg wallpaper, the tofi launcher, and wlr-randr for display settings, driven by an absolute virtio-tablet pointer. These are upstream ports, not custom code, and they are the part still being stabilized.

[ status ]

Work in progress, not yet fully stable. A substantial but unmistakable learning project. It boots under QEMU (TCG and KVM), runs bash and multithreaded userland, brings up the Wayland compositor, and drives real input / audio / network / storage devices. Rough edges remain: the desktop's client-presentation path and a few subsystems are still being hardened, so expect glitches there.
