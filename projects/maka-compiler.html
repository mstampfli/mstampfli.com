maka

A small statically typed language, optionals with flow-sensitive narrowing, fixed-size arrays, slices, references, and modules, compiled to C by a ~10k-line Python compiler. The outstanding, safe, and revolutionary programming language, if you grade generously.

the idea

Maka is a language built mostly to understand what a compiler actually guarantees. Static types catch a class of mistakes; optionals and flow-sensitive narrowing catch another. The point is to feel where those guarantees come from by writing every stage, lexer through C codegen, by hand, with no dependencies beyond Python 3.10+ and a C compiler.

The result is small and honest: about 10k lines of Python that turn a typed Maka source file into a single C file, link it against a tiny runtime, and hand it to gcc.

the language

Maka has primitives (int, float, bool, char, and unit), const-by-default bindings with explicit mut, and a syntax that leans C-adjacent. The flagship idea is optionals: ?T values are unwrapped with x! only after a flow-checked x? test, so the typechecker knows the unwrap is safe. The example below declares a data struct and a logic block, mutates an array element, and narrows an optional, it prints 7, 42, 7.

  • data / logic, data structs with per-field defaults; logic Name { ... } namespaces functions, called as Name.func(...) as a stand-in for methods.
  • struct literals, { x = 3, y = 4 } desugared against the expected type, including nested fields.

features

  • Optionals with flow narrowing, ?T with none, the check operator x?, and unwrap x!; unwrapping is flow-checked inside if (x?), while (x?), or a for condition. Optionals nest (??T) and lift implicitly through array elements, returns, and call arguments.
  • data & logic blocks, data declarations with field defaults; logic blocks grouping functions under a namespace.
  • Arrays & slices, fixed-size [3]int with literals and runtime bounds checking; slices ([]int, []Point) view arrays with indexing and mutability-checked element assignment.
  • References, non-null &T references, created with &x.
  • Mutability, const default and mut on variables and parameters; writes through non-mut bindings are rejected.
  • Strings & chars, string literals as char arrays ([3]char s = "hey"), char literals with the usual escapes.
  • Modules, one module per file, import with qualified access (Math.add), resolved recursively from the source directory.
  • Control flow & builtins, if/else, while, C-style for, break/continue/return, compound assignment, and the overloaded log builtin (no implicit newline).

pipeline

main.py drives a classic four-stage pipeline, all in compiler/.

  • Lexer (lexer.py), hand-written state machine producing tokens.
  • Parser (parser.py), recursive descent into an AST (ast.py), one module per file.
  • Typechecker (typechecker.py), checks all modules from the root, enforcing mutability, optional flow rules, struct-literal desugaring, and warnings (e.g. implicit int→float).
  • C codegen (codegen_c.py), emits a single C file; optionals become tagged structs, slices become pointer+length pairs, names are mangled per module.
  • cc, the generated C links against compiler/runtime.c (logging, panics) and is built with gcc.
[ status ]

Work in progress, developed in numbered phases (currently somewhere past phase 46). The pipeline is solid end to end for the features above; known rough edges live mostly in codegen for optional slices (?[]T). Error messages report character offsets, not line numbers. Use it to print integers with confidence; use it in production after the heat death of the universe.

demo.maka · prints 7, 42, 7
module demo;

data Point {
    int x = 0;
    int y = 0;
}

logic Geometry {
    int manhattan(Point p) {
        return p.x + p.y;
    }
}

unit main() {
    mut [3]int xs = [1, 2, 3];
    xs[1] += 5;

    ?int maybe = 42;
    if (maybe?) {
        log(maybe!);   // flow typing: unwrap allowed once checked
        log("\n");
    }

    Point p = { x = 3, y = 4 };
    log(Geometry.manhattan(p));
    log("\n");
}
compile + run
# emit C only
$ python3 main.py program.maka --out out.c

# also build a binary with gcc
$ python3 main.py program.maka --compile

# build and run it
$ python3 main.py program.maka --compile --run
7
42
7

notes

  • deps, Python 3.10+ and a C compiler, nothing else.
  • reserved, pub and embed are keywords, not yet implemented. No generics.
  • tests, 300+ .maka programs grouped by phase; _ok files compile, _err files must be rejected.
