veil

Most "route this through a VPN" setups are all-or-nothing and leak by default: every app shares one exit, and one chatty process can deanonymize the rest. Veil puts a single application inside its own Linux network namespace and forces every packet it sends through a chain you compose (WireGuard, OpenVPN, Tor, SOCKS5, HTTP), with a kill switch it verifies instead of assumes. Written in Go, with a CLI and a GTK GUI.

the idea

A normal VPN routes the whole host: every app shares one exit, and one misbehaving program can deanonymize everything else on the link. Veil flips that around. You give a single app its own network namespace and define exactly how its traffic leaves the machine, independent of whatever the rest of the host is doing.

isolation

  • Per-app network namespace, each profile spins up a dedicated Linux netns joined to the host by a veth pair. The target app launches inside it.
  • No bare-link path, the app sees only the tunnel, never the host's real interface or IP address.
  • DNS captured, name resolution is forced through the tunnel too, so there is no plaintext resolver leak around the side.

the chain

Exits are composed, not single. Stack hops in series, in any order, for real multi-hop:

  • Backends, WireGuard, OpenVPN, the Tor network, and SOCKS5 / HTTP proxies.
  • Multi-hop, for example host → WireGuard → Tor → app, so the WireGuard provider never sees Tor and Tor never sees your home IP.
  • Per-profile, every profile carries its own chain, so two apps can leave the machine through completely different paths at the same time.

a kill switch you can trust

Most kill switches add a firewall rule and hope. Veil verifies: before it hands the namespace to your app it actively checks there is no route to the internet except through the tunnel, and if the tunnel drops it fails closed. The app loses connectivity rather than spilling onto the bare link.

open core

The free edition is the full isolation engine, open source on GitHub: chains, the verified kill switch, profiles, the CLI and the GUI. A Pro edition adds an anti-detect / opsec stack for security research, and it is a drop-in, the same veil binary gated by a signed offline license:

  • Personas, per-profile browser identities and a forge that generates unique ones.
  • Fingerprint shaping, TCP, TLS and HTTP/2 characteristics aligned to the persona.
  • Pinned exits, lock a profile to a chosen exit endpoint, with launch-window and behavioral controls.
[ status ]

Work in progress, Linux-first. The free engine builds and runs, isolation, chains and the verified kill switch work; the Pro edition and its signed update channel are still being finished. A real but learning-grade tool, not a polished product release yet. For authorized, lawful privacy and security-research use only.
