maka-design

A warm, earthy, slightly retro-terminal design system: design tokens and mk- components in one stylesheet. Dark roasted-brown by default, with a paper-cream light variant, the same look reused across every mstampfli site, including the one you're reading now.

the idea

One look, defined once and reused across every site. maka-design is a single stylesheet, link it and you inherit the whole language. Everything derives from one token file, so you can recolor the entire system by editing a handful of custom properties: change the base palette and every card, button, table, and frame follows along. No per-page CSS, no theme drift.

foundations

  • Palette, deep roasted-brown surfaces and cream text, with a burnt-orange primary accent (#e0731d). No blues, no purples, no gradients.
  • The 2px frame, a warm-brown border on every framed element; on hover and focus it turns orange. That frame is the Maka signature.
  • Type, IBM Plex Mono for headings, labels, and buttons; IBM Plex Sans for body copy at 15px/1.65.
  • Texture, faint horizontal scanlines laid over the whole page at about 3% opacity, themed and opt-out-able.
  • Geometry, near-square radii (2/4/6px); depth lives in borders, not shadows.
  • Motion, 120ms ease color/border transitions only. No bounces, no fades.

theming

Components reference only semantic tokens, --mk-surface-1, --mk-border, --mk-accent, which alias a base palette (--mk-brown-*, --mk-orange-*). Dark is the default on :root; the paper-cream variant lives under [data-theme="light"] on <html>. To rebrand, edit the base palette; to retune a single role, edit an alias. Status tints are derived with color-mix() from one token each, no hand-picked tint colors exist.

this very page

The page you're reading is drawn entirely by maka-design. There's no framework and no build step, just one <link> to styles.css, the mk- classes, and the --mk-* tokens. The navbar, hero, spec strip, cards, code blocks, and footer are all stock components. It's the system describing itself.

[ ok ]

Want to see the paper variant? Toggle the [ light_mode ] switch in the nav and the whole system recolors live. Or open the kitchen sink to see every component on one page.

index.html · link it
<!-- one stylesheet, the whole look -->
<link rel="stylesheet"
      href="https://mstampfli.github.io/maka-design/styles.css?v=4">

<!-- then use mk- classes -->
<button class="mk-btn mk-btn--primary">get_started</button>

/* or reach for the tokens directly */
.callout {
  border: 2px solid var(--mk-border);
  color: var(--mk-accent);
}
