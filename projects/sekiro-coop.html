sekiro-coop

A seamless two-player co-op mod for Sekiro: Shadows Die Twice, written in Rust and injected as a DLL via me3. Sekiro ships with no multiplayer, so the networking, state sync, authority, and event replication are all built from scratch on top of function hooks into the live game process.

the idea

Sekiro has no native multiplayer. sekiro-coop adds seamless two-player co-op to v1.06 by injecting a Rust DLL into the running game and rebuilding everything a co-op session needs, transport, snapshot sync, authority, and game-event replication, on top of detours hooked directly into the game's own functions.

Scope. This is a mod for a single-player game, for personal and co-op use. The launcher runs Sekiro with Arxan disabled and online blocked, and it stays that way.

hooking

  • MinHook + AOB, MinHook is vendored in-tree; native function addresses are resolved by AOB scan at startup, so the mod finds the game's functions without hard-coded offsets.
  • Six live-validated detours, SetFlag, ApplyEffect, DeleteEffect, GiveItem, AddExperience, and WarpBonfire are hooked and exercised against a live Sekiro instance.
  • Hook-fed registry, the enemy registry is built from hook arguments (e.g. the entity pointer ApplyEffect receives) rather than sweeping WorldChrMan; pointer sweeps crashed on stale data, while hook-supplied pointers are known-good.
  • Trampoline-direct replay, replicated remote events are applied through trampoline calls to the original functions, so applying an event never re-enters the detour and echoes back over the wire.

data plane

  • PlayerSnapshot, full own-player state at 60 Hz, unreliable, bidirectional.
  • BridgeEvents, flag / effect / item / XP / warp events, edge-triggered (sent only on change) and reliable; application is gated behind SEKIRO_COOP_APPLY_REMOTE=1.
  • EnemyStates, host-only broadcast at 5 Hz, delta-filtered via a 64-bit per-entity digest so ~97% of entries are skipped on idle frames; the client resolves remote handles to local ChrIns pointers and applies decrement-only HP.
  • Desync detection, a 60-frame state-hash exchange with a three-strike session kill.
  • Reliability, sliding-window ACKs with sequence numbers, an ack bitmap, and a retransmit queue for packet types that need it; the session itself uses a magic + versioned header, handshake, and bincode-serialized bodies.

architecture

The build is a layered Cargo workspace, lowest to highest:

  • sekiro-sdk-sys, the memory substrate: AOB patterns, per-version offsets (V1_02, V1_06), pointer chains, native-function resolution, and live ChrIns readers/writers.
  • sekiro-sdk-core, typed entity wrappers, enums, character/item catalogs, and the MinHook FFI shim.
  • sekiro-sdk-bridge, the BridgeEvent dispatcher that translates hook callbacks into a typed event stream and back.
  • sekiro-coop-rollback, snapshot ring, state-delta compression, predictor, re-simulation, and the ChrInsStepper write-back path (unit-tested, not yet wired into live play).
  • sekiro-coop-authority, authority table (host owns enemies), proximity handoff with hysteresis, and deterministic seeded PCG RNG.
  • sekiro-coop-net, UDP transport, session/handshake, wire format, reliability, and the desync detector.
  • sekiro-coop-dll, the injected cdylib: DLL entry, detour install, the 60 Hz fallback ticker, and an optional hudhook ImGui overlay.
  • sekiro-coop-emevd, a structured EMEVD binary reader/writer with a 386-instruction catalog and custom-event injection.
[ status ]

The data plane is validated end-to-end against a live Sekiro instance and a peer-simulator standing in as the second player, but the mod is not yet end-user playable. The open blocker is rendering the remote player: writing positions into ChrIns memory is accepted but ignored by the render path, and the natives that could drive a puppet NPC have no known AOBs. The planned unblock is patching common.emevd so the game's own event VM dispatches those natives, that EMEVD-patch pipeline is still pending.

build + launch (windows)
# build the mod DLL
PS> cargo build --release \
       -p sekiro-coop-dll
# -> target\release\sekiro_coop.dll

# offline tools (peer-sim, emevd…)
PS> cargo build --release \
       -p peer-simulator

# preflight + build + stage + me3 launch
PS> .\launch.ps1 -Peer host
PS> .\launch.ps1 -Peer client
env
SEKIRO_COOP_PEER      # host|client
SEKIRO_COOP_BIND      # 0.0.0.0:28000
SEKIRO_COOP_PEER_ADDR # ip:port
SEKIRO_COOP_APPLY_REMOTE # 1
