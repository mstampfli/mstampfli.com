voxelg

A real-time voxel engine in Rust where the entire world is rendered by a WGSL compute shader. There is no triangle geometry for terrain, each frame, rays are marched through a three-level occupancy-bit pyramid directly on the GPU, and a fullscreen-triangle blit presents the result.

the idea

Skip the triangles. Instead of meshing terrain, voxelG marches rays through a three-level occupancy-bit pyramid, chunk → tile → brick, entirely inside a WGSL compute shader, then blits the result with a fullscreen triangle. The same binary runs solo, as a headless server, or as a multiplayer client, built on wgpu 23 and winit 0.30.

rendering

  • Hierarchical DDA, a 3-level bit pyramid: one bit per 16³ tile in each 64³ chunk, one bit per 4³ brick in each tile, and a 64-bit occupancy mask per brick, with exact cell-boundary snapping.
  • Beam pre-pass, at 1/8 resolution, one ray per 8×8 pixel block walks the world at tile granularity and writes first-hit depth to an r32float texture; the main pass fast-forwards to that depth minus a 16-voxel safety margin.
  • Uniform-region compaction, CPU passes detect 4³ bricks and 16³ tiles filled with a single opaque material and publish packed byte LUTs, so the DDA terminates at the cell's entry face without descending the hierarchy.
  • Temporal dirty-tile, dirty bricks from physics and edits are projected into 8×8 screen tiles on the CPU; the compute shader early-outs on clean tiles and both compute passes are skipped when nothing changed.
  • Distance LOD, beyond 400 voxels the DDA terminates at brick granularity, shading with the brick's topmost solid material so distant terrain keeps its surface colour.

world & physics

  • Gerstner water, four-wave normals from closed-form height-field derivatives, Schlick Fresnel mixing a traced reflection with a Snell-refracted trace (η = 1/1.33), Beer, Lambert absorption, shoreline foam, caustics and a submerged absorption post-effect.
  • Dispersive glass & TIR, Fresnel reflection plus per-channel refraction for chromatic dispersion (n = 1.48/1.50/1.52), total-internal-reflection fallback, and distance-compounding tint.
  • Volumetric clouds, slab-raymarched cumulus (fbm body under a coverage mask, 3 cone samples toward the sun for self-shadowing, Henyey, Greenstein scattering) plus accumulated god rays.
  • Foliage sway, sub-voxel cube leaves, tall grass as up to 22 tapered bowed blades per voxel and crossed-plane flowers, all driven by one global wind field with per-face edge sway.
  • Cellular-automaton sand / water / smoke, physics at 30 Hz: 8-level water level propagation, bitmask sand gravity and rising smoke, iterating only "active" bricks.

multiplayer

  • Lock-free TCP, length-prefixed bincode messages; each connection gets a reader and a writer thread bridged to the single-owner game thread through crossbeam's lock-free MPMC channels, no shared mutexes.
  • Seed sync + edit log, world state syncs by shared seed plus a persistent edit log replayed to joiners; sphere destruction travels as a single Explode message expanded locally on each client.
  • Interest management, pose updates capped at 20 Hz fan out with distance-based interest (600-voxel radius); remote players render as ray-traced colour-hashed boxes.
[ status ]

Experimental graphics playground, not a game. Terrain is seed-deterministic; the server keeps its edit log in memory only (no on-disk persistence), and remote players are placeholder markers. Constants are tuned by eye on a single machine, expect to adjust them for yours.
