cve2detect

Turn a CVE into a head start on defense. Give it a CVE id and it pulls the structured advisory from OSV and NVD, then generates a Sigma rule, a Nuclei template, per-ecosystem version checks, and a pinned repro scaffold, all seeded with the real affected packages.

what it does

Triage first. cve2detect enriches the advisory with what a defender actually decides on: whether it is on CISA's KEV list (exploited in the wild), its EPSS exploitation probability, whether a public exploit is referenced, and the CVSS/CWE. So before writing a line of detection you know whether you even need to. Then it generates artifacts seeded with that data: a Sigma rule, a Nuclei template, per-ecosystem version checks, a CWE-driven Semgrep rule for code-level bugs, and a repro pinned to a vulnerable version. (OSV's CVE record only has git-commit ranges, so it follows the GHSA/PYSEC/RUSTSEC aliases for the real packages.) An optional --ai flag drafts a concrete detection and repro from the whole enriched context through the Claude CLI, no API key.

live demo

Type a CVE id. This runs entirely in your browser against OSV, EPSS, and nuclei-templates, showing the EPSS exploitation probability, the affected packages, and the real community Nuclei detection (the actual runnable YAML) when one exists, live. Try CVE-2021-44228 (Log4Shell) or CVE-2021-23337 (lodash).

what the full CLI adds

  • KEV + exploit intelligence, CISA Known-Exploited status and EPSS probability, plus concrete public-exploit sources: Metasploit modules and Exploit-DB entries, on top of CVSS/CWE.
  • A CWE-driven Semgrep rule for code-level bugs (SQLi, XSS, RCE, deserialization, ...), language-matched to the affected package.
  • A Nuclei template seeded from the references, plus the Sigma rule.
  • A pinned repro scaffold, a ready-to-run project (npm / pip / cargo) wired to a vulnerable version with a TODO trigger.
  • --ai, drafts a concrete detection and repro from the full enriched context via the Claude CLI.
usage
python3 cve2detect.py CVE-2021-44228
python3 cve2detect.py CVE-2021-23337 --ai
