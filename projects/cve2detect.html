cve2detect

Turn a CVE into a head start on defense. Give it a CVE id and it pulls the structured advisory from OSV and NVD, then generates a Sigma rule, a Nuclei template, per-ecosystem version checks, and a pinned repro scaffold, all seeded with the real affected packages.

what it does

OSV's CVE record usually only has git-commit ranges, so cve2detect follows the GHSA / PYSEC / RUSTSEC aliases to resolve the real ecosystem packages and version ranges, and pulls CVSS and CWE from NVD. From that it writes a Sigma detection skeleton, a Nuclei web-check template, per-ecosystem version checks, and a minimal repro pinned to a vulnerable version. The generated rules are honest skeletons seeded with real metadata, so you start from structure, not a blank file. An optional --ai flag drafts fuller artifacts through the Claude CLI, no API key.

live demo

Type a CVE id. This runs entirely in your browser against the public OSV API and generates the artifacts live. Try CVE-2021-44228 (Log4Shell) or CVE-2021-23337 (lodash).

Results appear here.

what the full CLI adds

  • NVD enrichment, CVSS score and CWE class on top of the OSV package data.
  • A Nuclei template seeded from the references, plus the Sigma rule you see in the demo.
  • A pinned repro scaffold, a ready-to-run project (npm / pip / cargo) wired to a vulnerable version with a TODO trigger.
  • --ai, drafts a concrete detection idea and a repro outline via the Claude CLI.
usage
python3 cve2detect.py CVE-2021-44228
python3 cve2detect.py CVE-2021-23337 --ai
