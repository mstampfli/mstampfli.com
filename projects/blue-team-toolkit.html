blue-team-toolkit

A menu-driven defensive-security toolkit for rapidly assessing, hardening, and monitoring Windows and Linux hosts. One workflow wraps a curated set of well-known DFIR, hardening, and recon tools, install → map → recommend → harden → triage → hunt, and rolls the result into a single graded security report card. For authorized use only.

the idea

Defending a system is the final exam for understanding it. To harden a host you first have to know what it is, what it exposes, and what already looks wrong, so the toolkit wraps that whole loop behind a single menu. Each platform has its own entry point, but the Windows and Linux sides mirror each other in structure and output format, so the same workflow carries across both.

[ scope ]

This is defensive (blue team) tooling intended for authorized use only: systems you own, or are explicitly cleared to assess and defend (production hardening, lab work, CTFs, incident response). Some bundled tools (nmap, NetExec, nuclei, LinPEAS/WinPEAS, SharpHound) are dual-use; run them only against in-scope assets.

structure

Two entry points launch identical menu structures: prep.ps1 (Windows console menu) and prep.sh (Linux whiptail menu). Both create an output/ tree (reports, logs, findings.jsonl, facts.json) and a tools/ directory next to the script; per-platform logic lives in lib/windows/*.ps1 and lib/linux/*.sh, kept at feature parity.

the workflow

  • install, the installer reads tools.json and installs every tool, or a custom selection, via winget, apt, pipx/pip, direct URL, git clone, and GitHub-release asset matching.
  • map, a discovery pass writes facts.json: OS identity, role detection, security posture, listening ports with bind-interface exposure analysis, privileged services, firewall posture, and which bundled tools are present. Optional heavy passes add WinPEAS/autoruns/nmap or lynis/linpeas/nmap.
  • recommend, reads facts.json and prints a prioritized P0/P1/P2 hardening punch list with the matching hardening step, plus tools present vs. missing.
  • harden, a semi-automated checklist, dry-run by default, with per-step confirmation, a real file backup before each edit, and a rollback hint for every change; steps that remove functionality are gated and labeled.
  • triage, per-host raw-state collection to a timestamped report; dispatches installed scanners first, then dumps processes, sockets, persistence, accounts, and live state.
  • hunt, a backdoor/persistence hunt that dispatches available hunters (autoruns/chainsaw/hayabusa/hollows_hunter/loki), then runs high-specificity "CLEAR" checks, separates noisier "REVIEW" exposure checks, and compares against the saved baseline.
  • report card, rolls facts.json and the accumulated findings.jsonl into one graded report, an A-F posture grade, prioritized issues, and open findings, as a self-contained HTML page plus Markdown.

modules

The per-platform menu maps onto a shared set of library modules; both sides are at feature parity.

  • installer, manifest-driven install across multiple package managers and download methods.
  • map, host discovery into facts.json; roles, ports, posture, and tool inventory.
  • recommend, turns the facts into a prioritized P0/P1/P2 hardening punch list.
  • harden, Windows: LSA RunAsPPL, disable SMBv1, SMB signing, the SMBGhost workaround, Print Spooler, LLMNR/NetBIOS/WPAD, NTLMv2-only, PowerShell logging, audit policy, a persistence baseline snapshot. Linux: SSH lockdown, auditd with the Neo23x0 ruleset, fail2ban, UFW default-deny, sysctl hardening, the PwnKit pkexec workaround, and package/service baselines.
  • triage, dispatches WinPEAS/autoruns/sigcheck or lynis/linpeas/rkhunter/chkrootkit, then collects live state.
  • hunt, recent accounts, services/tasks in writable paths, masquerading processes, Defender tamper/exclusions, suspicious listen ports, privileged-group changes, WMI/AppInit persistence, web-shell paths, and baseline drift.
  • recon, air-gap-aware network recon. INTERNAL: nmap discovery + service scan + NetExec SMB enum + nuclei against web hosts. EXTERNAL: Shodan InternetDB, crt.sh subdomain pull, DNS enum, and NAT-hairpin nmap, into an editable recon-targets.txt.
  • findings, a viewer for the persistent findings.jsonl accumulated across runs; summary, recent, by-type, and deduped CSV export.
  • report card, aggregates the map facts and accumulated findings into an A-F graded HTML + Markdown report, the one-glance view of where the host stands.
[ status ]

Single-author project; the Windows and Linux sides are at feature parity. No automated tests. The hardening module is dry-run by default and backs up every file it touches, but review its output before applying changes to any host you care about, and run the dual-use tools only against in-scope assets.

requirements

  • Linux, bash with whiptail, jq, curl (checked at startup); Debian/Ubuntu-family assumed. Hardening and parts of triage use sudo.
  • Windows, Windows PowerShell 5.1+; the hardening module requires an elevated (Administrator) session and refuses to run otherwise.
  • tools, bundled third-party tools download into tools/ on demand; nothing is committed to the repo.
