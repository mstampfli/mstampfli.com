maka

A systems language that puts in the type what C leaves implicit: who owns each piece of memory, and whether a pointer can be null. On top of that foundation it adds structural attr/has traits, pattern-matched enums, user-defined iterators, lightweight concurrency the compiler checks, and inline C and Rust interop. It compiles to portable C you can read, the compiler is written in Rust, and the standard library is written in Maka itself.

the idea

C makes you carry two facts in your head that the compiler never sees: who is responsible for freeing a pointer, and whether it can be null. Maka moves both into the type. A pointer's kind says how the memory behaves, and you cannot dereference a nullable one without proving it is non-null right there. Everything else, traits, enums, iterators, concurrency, follows from wanting those guarantees without giving up a small, readable language.

It stays honest about its target: a Maka file becomes a single portable C file that your system C compiler builds. The compiler, makac, is a Rust workspace, and the standard library (Option, Result, strings, vectors) is written in Maka and shipped inside it.

pointers tell you what they are

Five pointer kinds, each with one job, so the type says how the memory behaves:

  • &T / &mut T, tracked borrows, never null.
  • *T, nullable, non-owning.
  • own *T / own &T, heap owners, nullable or strict.
  • raw *T, the escape hatch for what the compiler can't see, C interop, fixed addresses, unsafe code.

alloc value is the only allocator, and its result must land in an owning slot. A move-and-lifetime checker tracks owners and borrows: use-after-move, returning a local borrow, or stashing a borrow that outlives its owner are all compile errors.

forced handling, no runtime null checks

You cannot dereference a nullable pointer without proof of non-null at the deref site: a flow-narrowing window, an early-exit guard, or an immediate value straight from alloc. There is no unwrap macro and no hidden runtime null check, the proof is a compile-time obligation.

traits without the boilerplate

  • attr + has, structural traits instead of traits-plus-impl. An attr declares a contract (with optional default bodies); Type has Attr { ... } provides the implementation, with _ as the placeholder for the implementing type.
  • parametric receivers, an impl's receiver can be a shape, not just a concrete type: *T has Foo, int has Foo, Box<T> has Foo, Result<T, E> has Foo. First-order structural unification at the call site, plus coherence-as-overlap-rejection (no specialization, no backtracking) so dispatch is unambiguous.
  • associated types, an attr can declare type Slot; and each impl supplies type Slot = Concrete;; type expressions like T::Slot resolve at monomorphization. data Wrap<T: Stored> { T::Slot inner; } materializes a different concrete shape per T. Defaults (type Slot = int;) and bounds (<T: Foo<Slot = i64>>) both ship.
  • generics over attrs, fn render<T: Show>(&T x), plus overloading and dynamic dispatch where you want it.
  • inline frame-splicing, an inline fn splices into the caller's frame: propagate X inside it returns X from the outer function. The stdlib's try_ok inline helper composes this with Result<T, E> for an early-exit-on-Err idiom that needs no language change.

enums, matching, and iteration

  • tagged enums + match, payload-carrying variants destructured in match arms; the stdlib's Option and Result are ordinary Maka enums.
  • user iterators, implement next in a logic block and your type drives for (x in it) directly.
  • closures + operator overloading, first-class lambdas and overloadable operators for value-like types.

concurrency the compiler checks

Threads are spawn / join; values cross a thread boundary only through transfer (move) or share (shared reference), and the compiler enforces the sendable/shareable bound at the crossing site rather than trusting you to get it right.

Underneath sits a portable runtime: ucontext-based cooperative fibers for cheap in-thread tasks, a work-stealing job pool for CPU-bound parallelism, and pthread-backed sync primitives (mutex, waitgroup, once, channels) with destroy handshakes that wait for in-flight parkers before tearing the primitive down. Wake / cancel / close cross thread boundaries through per-scheduler inboxes; AIO work runs on detached worker threads with the calling fiber gated by an in-flight flag so cancel can't free a buffer the worker is still writing to.

freestanding mode

One flag, --freestanding, strips the libc-using prologue, skips the auto-include of std.maka, and routes alloc / free / panic / log to user-supplied extern symbols (__maka_alloc / __maka_free / __maka_panic / __maka_log_int / __maka_log_str). The emitted C compiles cleanly under gcc -ffreestanding -nostdlib — no stdio, no malloc, no pthread — and the OS author links it with their own bootloader and runtime. The atomic builtins (atomic_load, atomic_cas, atomic_fetch_*, atomic_fence) keep working because they lower to gcc/clang's __atomic_* intrinsics — no C runtime, no syscalls. A 20-line stub runtime in the test suite proves the path end-to-end. The same compile-to-C strategy serves both hosted and kernel targets by swapping the prologue.

modules, and inline C and Rust

  • explicit modules, every cross-module item needs an import; the error at any unimported reference names the exact import line to add. pub controls visibility.
  • C inline, cinclude / cblock drop C in directly; raw *T and the linker flags carry the ABI.
  • Rust inline, an rblock holds real Rust and rdep adds crates; each pub fn becomes a callable Maka function, #[repr(C)] structs mirror to data, and anything else rides as an opaque Rust<T> handle that participates in ownership. Send/Sync is verified at thread crossings.

where it stands

Real and exercised: the compiler passes a suite of 174 end-to-end programs (each checked against expected output) plus a growing set of negative tests that must fail to compile — so the ownership, null-handling, trait, and interop rules are enforced, not aspirational. The generated C builds with any recent gcc, clang, or mingw-w64.

Cross-platform parity is verified by running the full suite on both targets at every change: 174 / 174 on Linux and 155 / 155 on Windows (the nineteen-test delta is platform-specific paths — inotify, signalfd, eventfd — that have Windows shims but no behavioral test on the Windows side). The freestanding mode is smoke-tested separately: Maka → no-libc C → object → static ELF with a stub kernel runtime. macOS isn't a CI target yet — the runtime is written and adversarially audited end-to-end for it (sockaddr sin_len / sun_len, os_unfair_lock, kqueue per-thread fds, no Linux-only syscalls in mac paths), pending a run on real hardware.

parametric receivers + associated types
attr Cell {
    type Repr;                    // each impl picks the cell type
    int probe(&_ self);
}

// pointers: one impl covers every *T, T binds at the call site
*T has Cell {
    type Repr = *T;
    int probe(&*T self) { return 100; }
}

// primitives can be has-implementors too
int has Cell {
    type Repr = int;
    int probe(&int self) { return self + 1000; }
}

// the placeholder data pattern: T::Repr is resolved per concrete T
data Wrap<T: Cell> { T::Repr inner; }

unit main() {
    Wrap<int> w = { inner = 42 };    // materializes as { int inner; }
    log(w.inner);                       // 42
    int a = 5;
    log(probe(&a));                    // 1005, from `int has Cell`
}
pattern matching on enums
enum Shape {
    Circle { float radius },
    Rect { float w, float h },
    Empty,
}

float area(Shape s) {
    return match (s) {
        Circle{radius} radius * radius * 3.14,
        Rect{w, h}     w * h,
        Empty          0.0,
    };
}
ownership through an inline callee
data Box { int v; }

inline int take(own *Box b) {
    if (b == null) { return -1; }
    return b!.v;     // proof comes from the guard
}

unit main() {
    own *Box b = alloc Box { v = 42 };
    log(take(b));            // 42, freed at splice exit
}
inline Rust, called from Maka
rdep regex = "1";          // a cargo dependency

rblock "
    use regex::Regex;
    pub fn count(hay: &str, pat: &str) -> i32 {
        Regex::new(pat)
            .map(|r| r.find_iter(hay).count() as i32)
            .unwrap_or(-1)
    }
";                          // each pub fn -> a Maka fn

unit main() {
    log(count("a1b2c3", "[0-9]"));   // 3
}
compile + run
$ makac hello.maka -o hello --run
hello, world

the compiler

  • lexer → parser → sema → codegen, a Rust workspace; sema does resolution, type-check, and the lifetime/move pass.
  • emits one portable C file, built by your system gcc/clang.
  • stdlib in Maka, Option, Result, strings and vectors, compiled in.
