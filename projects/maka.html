maka

A from-scratch systems language with explicit ownership, structural traits via attr/has, and inline functions that propagate errors through the caller's frame. It compiles to portable C. This is a ground-up redesign, the compiler rewritten in Rust, of an earlier Python implementation of the language.

the idea

Maka started as a Python compiler for a small typed language. This is the second take: the same language taken seriously, redesigned around explicit ownership and rebuilt from scratch with the compiler written in Rust. The goal is the same as before, to feel exactly where a language's guarantees come from by writing every stage by hand, but the guarantees are stronger now: who owns memory, and whether a pointer can be null, are both in the type.

It stays honest about its target: a Maka source file becomes a single portable C file, compiled by your system C compiler. The compiler is makac.

pointers tell you what they are

There are five pointer kinds, each with one job, so a type says how the memory behaves:

  • &T / &mut T, tracked borrows, never null.
  • *T, nullable, non-owning.
  • own *T / own &T, heap owners, nullable or strict.
  • raw *T, the escape hatch for what the compiler can't see, C interop, fixed addresses, unsafe code.

alloc value is the only allocator, and its result must land in an owning slot.

forced handling, no runtime null checks

You cannot dereference a nullable pointer without proof of non-null at the deref site: a flow-narrowing window, an early-exit guard, or an immediate value straight from alloc. There is no unwrap macro and no hidden runtime null check, the proof is a compile-time obligation.

attr + has, and inline propagation

  • attr + has, structural traits instead of traits-plus-impl. An attr declares a contract (with optional default bodies); Type has Attr { ... } provides the implementation, with _ as the placeholder for the implementing type. Generic functions bound over attrs, fn render<T: Show>(&T x).
  • inline frame-splicing, an inline fn splices into the caller's frame: propagate X inside it returns X from the outer function, not from the inline callee. Useful for shaped early-exit; recursion among inline functions is forbidden.
  • data structs, data Point { mut int x; mut int y; } with struct literals desugared against the expected type.
[ status ]

Work in progress, and the active line of development for the language (it supersedes the older Python compiler). The ownership model, forced null handling, attr/has, generics, and inline propagation are the design center; expect rough edges and missing corners while it settles.

traits.maka · prints "point"
attr Show {
    string label(&_ self) { return "any"; }   // default body
}

data Point { mut int x; mut int y; }

Point has Show {
    string label(&Point self) { return "point"; }
}

unit render<T: Show>(&T x) {
    log(x.label());
}

unit main() {
    Point p = { x = 3, y = 4 };
    render(&p);                // "point"
}
ownership through an inline callee
data Box { int v; }

inline int take(own *Box b) {
    if (b == null) { return -1; }
    return b!.v;     // proof comes from the guard above
}

unit main() {
    own *Box b = alloc Box { v = 42 };
    log(take(b));            // 42
}
compile + run
$ makac hello.maka -o hello --run
hello, world
