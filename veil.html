veil▮

Per-application network isolation. Route any single app through a tunnel chain of its own (WireGuard, OpenVPN, Tor, SOCKS5, HTTP), inside a dedicated Linux network namespace, behind a kill switch that is verified, not assumed.

why / one app, one tunnel

Isolation that scopes to the app, not the host

A conventional VPN routes the whole machine: every app shares one exit, and one chatty process can deanonymize the rest. Veil scopes the tunnel to a single application.

Per-app isolation

Each profile runs its target process in a dedicated Linux network namespace. The app sees only the tunnel, never the host's real interface, IP, or resolver.

Composable multi-hop

Chain WireGuard, OpenVPN, Tor, SOCKS5 and HTTP hops in any order, for example host → WireGuard → Tor → app, so no single provider sees both ends.

Verified kill switch

Veil confirms there is no route to the internet except the tunnel before the app starts, and fails closed if the tunnel drops. It verifies, it does not assume.

editions / free and pro

Two editions, one binary

Veil is open-core. The complete isolation engine is free and open source; the anti-detect and operational-security stack is a commercial, drop-in upgrade, the same binary unlocked by a signed offline license.

Get veil

Veil Core is free and open source. Linux is the primary target today, build and install in one step. Prebuilt signed binaries and Windows / macOS builds are in early access.

install (linux)
$ git clone https://github.com/mstampfli/veil
$ cd veil && ./install.sh
[ ok ] built veil, veil-gui, veil-bridge
[ ok ] installed to /usr/local/bin
$ veil up myprofile

platforms

  • Linux, x86-64, the primary target. Build from source today.
  • Windows / macOS, builds in progress.
